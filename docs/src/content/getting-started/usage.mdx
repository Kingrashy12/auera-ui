<Details id='theme'>
 <HeaderLabel id="theme" as='h3'>Theme</HeaderLabel>
 <p className="text">AueraUI allows you to customize the look and feel of your application with its flexible theming system. 
 The <CodeTag>ThemeProvider</CodeTag> component enables global theme management, allowing you to apply consistent styling across your app.</p>
</Details>

<Details id="default-theme">
 <HeaderLabel id="default-theme" as='h3'>Default Theme</HeaderLabel>
<p className="text">By default, AueraUI detects the user's system theme and applies the appropriate theme. This provides a seamless experience for users,
 as the UI will automatically align with their system settings.</p>
</Details>

<Details id="toggle-theme">
 <HeaderLabel id="toggle-theme" as='h3'>ToggleTheme</HeaderLabel>
<p className="text">
AueraUI provides a toggle function that allows you to manually switch between the user's system theme and AueraUI's theme set.
 The toggle function has two main options:
 </p>

 <ul className="text font-inter list-disc pl-3 gap-4 flex flex-col">
    <li>
        <CodeTag>toggleTheme.system</CodeTag>: Switches to the user's system theme based on the user's operating system settings.
    </li>
    <li>
        <CodeTag>toggleTheme.main</CodeTag>: Switches to AueraUI's predefined theme set, 
    </li>
  </ul>
  <p className='text'>
  This toggleTheme functionality is useful when you want to give users the ability to manually switch between system preferences and the app's theme.
  </p>
  <h4 className="text-primary font-inter font-semibold">
  Example Usage:
  </h4>
  <CodeBlock fileName='App.tsx' lg='tsx' code={`import { useTheme, Button } from 'auera-ui';

export default function App() {
  const { toggleTheme } = useTheme();

  return (
    <div>
      <Button onClick={() => toggleTheme.system()}>Use System Theme</Button>
      <Button onClick={() => toggleTheme.main()}>Use AueraUI Theme</Button>
    </div>
  );
}`} />
</Details>

<Details id="trigger">
  <HeaderLabel id="trigger">Trigger</HeaderLabel>
  <p className="text">Triggers in AueraUI are a set of components designed to let you open specific UI elements or components
   from anywhere within your application seamlessly.</p>
  <p className="text">Most components in AueraUI are designed to integrate seamlessly with Triggers, making it easy to
   create dynamic and interactive user interfaces.</p> 
</Details>

<Details id="using-trigger">
  <HeaderLabel id="using-trigger" as='h3'>Using Trigger</HeaderLabel>
<p className="text">There are two ways to use a component trigger in AueraUI:</p>
<ul className="text font-inter list-disc pl-3 gap-4 flex flex-col">
  <li>
    <b>Using component-based triggers</b>: These are specialized components like <CodeTag>ModalTrigger</CodeTag> and
     <CodeTag>DrawerTrigger</CodeTag>, designed to directly open specific components.
  </li>
  <li>
    <b>Using the <CodeTag>withTrigger</CodeTag> prop</b>: This prop is available for select components in AueraUI, such as
     <CodeTag>Button</CodeTag> and <CodeTag>IconButton</CodeTag>, enabling them to act as triggers for specific components.
  </li>
</ul>
</Details>

<Details>
<p className="text">
  Refer to the <TabLink to="/docs/components/button#with-trigger" label="Button" /> documentation for a detailed usage example
   of the <CodeTag>withTrigger</CodeTag> prop.
</p>
</Details>

<Details id="trigger-props">
  <HeaderLabel id="trigger-props" as="h3">Trigger Props</HeaderLabel>
  <p className="text">
    When using the <CodeTag>withTrigger</CodeTag> prop, there are a set of required props that need to be provided:
  </p>
  <ul className="text font-inter list-disc pl-3 gap-4 flex flex-col">
    <li>
      <b><CodeTag>trigger</CodeTag></b>: Specifies the target component to trigger, such as <CodeTag>Modal</CodeTag> or <CodeTag>Drawer</CodeTag>.
    </li>
    <li>
      <b><CodeTag>triggerType</CodeTag></b>: Determines the type of trigger action, either <CodeTag>open</CodeTag> or <CodeTag>close</CodeTag>.
    </li>
    <li>
      <b><CodeTag>triggerValue</CodeTag></b>: A unique value used to identify the target component. This value must match the corresponding
       value provided to the parent or "barrel" component (e.g., <CodeTag>Modal</CodeTag> or <CodeTag>Drawer</CodeTag>).
    </li>
  </ul>
</Details>
